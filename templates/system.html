System Architecture

A breakdown of the Convolutional Neural Network (CNN) pipeline and calorie estimation logic.

Data Processing Pipeline

1

Image Acquisition

User uploads an image via the web interface. System validates format (JPG/PNG).

2

Pre-processing

Image resizing (224x224), normalization, and formatting for the model.

3

CNN Classification

MobileNetV2/ResNet50 model extracts features and predicts the Kuih class.

4

Calorie Estimation

Predicted class maps to KKM nutritional database for calorie retrieval.

Core Technology

  • Deep Learning Model

    Utilizes Transfer Learning with MobileNetV2 or ResNet50, pre-trained on ImageNet and fine-tuned on the Kuih dataset.

  • Data Source

    Authoritative calorie data sourced from the Ministry of Health Malaysia (KKM) and MyNutriDiari.

Enhancements

  • Gemini AI Integration

    Generative AI provides cultural context, descriptions, and verifies caloric ranges for recognized items.

  • MongoDB Atlas

    Cloud database integration for storing scan history and tracking user feedback for model improvement.
